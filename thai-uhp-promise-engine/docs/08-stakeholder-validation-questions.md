# Stakeholder Validation Questions

Use these if a Thailand auto service chain, Michelin contact, tire distributor, or fleet/service operator engages. The goal is to validate the operational pain, not pitch vague AI.

## Problem validation

1. How often do premium/UHP customers fail to get a same-day or next-day fit because the exact SKU is unavailable?
2. Which UHP sizes are hardest to fulfill outside Bangkok?
3. Do branches track lost sales when the customer declines a delayed fitment?
4. Are staggered fitments a common source of branch-level mistakes?
5. Which regions create the most painful premium tire fulfillment cases: Isaan, North, South, Eastern Seaboard, or Bangkok suburbs?

## Inventory and transfer validation

1. Can branches see real-time stock across other branches and DCs?
2. Is stock reservable at branch level before the customer arrives?
3. What is the normal transfer path for rare UHP SKUs outside Bangkok?
4. What are the practical cut-off times for overnight transfer?
5. How often does inventory show stock that is already reserved, damaged, or otherwise unavailable?

## Service capacity validation

1. Are bay slots visible centrally?
2. Can premium tire jobs be reserved as appointment slots?
3. Does fitting a Lamborghini/Porsche/AMG require specific equipment, branch capability, or senior technician approval?
4. Are some branches marked as UHP-capable or premium-vehicle-capable?

## Data access validation

1. Which systems hold SKU master, stock, reservations, appointments, and transfers?
2. Can anonymized inventory snapshots be exported for a prototype?
3. Can historical delivery timestamps be exported by origin/destination?
4. Can lost-sales or quote-request data be exported?
5. Are there restrictions on using Google Maps style routing data in internal tools?

## Success metric validation

1. What is more valuable: reducing stockouts, reducing overstock, improving customer promise accuracy, or increasing UHP conversion?
2. What would make a branch advisor trust the recommendation?
3. What evidence would an operations leader need before piloting this?
4. What is the smallest pilot region that would be operationally meaningful?

## Strongest pilot proposal

Pilot the system as a decision-support layer for rare UHP requests in:

- Bangkok premium branches
- Khon Kaen / Udon Thani / Nakhon Ratchasima branches as Isaan coverage points
- One Bangkok-area DC or premium stock hub

The pilot should measure promise accuracy, premium tire conversion, and avoided failed appointments.
